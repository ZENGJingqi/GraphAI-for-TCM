# Data Dictionary

This document summarizes the purpose and schema of the main files in the `Data` directory so that users can understand how the released data support the prediction and training workflows.

## 1. Editable user input

### `Test_input.xlsx`

This is the recommended template for custom prediction input. The released workbook contains one sheet, `Sheet1`, with the following columns:

- `CPM_ID`: formula identifier used to group rows belonging to the same prescription or formula instance
- `CHP_ID`: identifier of the Chinese herbal piece
- `Dosage_ratio`: relative dosage value used as an extra node-level feature during graph construction
- `Chinese_herbal_pieces`: human-readable herb name for inspection and editing convenience

Each row represents one herb entry inside a formula. Multiple rows with the same `CPM_ID` define one complete formula graph.

## 2. Reference tables

### `CHP_Encoder.tsv`

This file provides the base numeric feature vector for each Chinese herbal piece (`CHP_ID`). The released table contains:

- source indicators and source principal components, such as `Sources_Viridiplantae` and `Sources_PC1` to `Sources_PC10`
- medicinal-property embedding dimensions, `Medicinal_properties_1` to `Medicinal_properties_15`
- compatibility-related embedding dimensions, `Combination_1` to `Combination_30`
- efficacy-related embedding dimensions, `Efficacy_1` to `Efficacy_30`

In the graph-construction notebook, the dosage ratio from the input workbook is appended to these encoded features, producing a 91-dimensional node feature vector in the released graph dataset.

### `CHP_Medicinal_properties.tsv`

This table maps each `CHP_ID` to medicinal-property annotations used to create virtual semantic nodes and herb-to-property edges during graph encoding. The main columns are:

- `CHP_ID`
- `Medicinal_properties`
- `Class`
- `x_rank`
- `y_rank`

The `Class` field is used in the notebook to connect actual herb nodes to higher-level medicinal-property categories such as medicinal flavor or meridian tropism.

### `Chinese_herbal_pieces.tsv`

This table is the identifier mapping and interpretation layer for herbs. The main columns are:

- `CHP_ID`
- `Chinese_herbal_pieces`
- `Chinese_synonyms`
- `Pinyin_term`
- `English_term`
- `Sources`

This file is useful when converting model outputs back into interpretable herb names and multilingual labels.

## 3. Generated inference files

### `all_graphs_to_be_predicted.pt`

This file is generated by `Python/1_Graph Embedding in TCM Formulas.ipynb`. It stores PyTorch Geometric graph objects created from the rows in `Test_input.xlsx`.

### `prediction_outputs.tsv`

This file is generated by `Python/2_Prediction Using the GAT Model.ipynb`. It contains:

- `cpm_id`
- five output dimensions, `Class_1` to `Class_5`
- sixty-four hidden graph representation dimensions, `hg_1` to `hg_64`

### `attention_weights.tsv`

This file is generated by the GAT prediction notebook and records edge-level attention weights. Its main columns are:

- `cpm_id`
- `Source`
- `Target`
- attention-head outputs such as `attn_weights_1_head_1` to `attn_weights_3_head_1`

It captures how the attention model distributes weight across herb-property and herb-herb relations.

### `attention_averages.tsv`

This file stores averaged attention summaries derived from `attention_weights.tsv`.

### `calculated_attention_weights.tsv`

This file contains a simplified compatibility-oriented table with the columns:

- `cpm_id`
- `Source`
- `Target`
- `attention`

It is used in the downstream quantitative interpretation and heatmap workflow.

In the current released example run, the bundled TSV output files were regenerated from the released `Test_input.xlsx` example set so that the input and output assets remain consistent.

## 4. Training datasets

### `all_graphs_with_labels-train.pt`

This is the graph training dataset used by `Python/Graph Attention Network.ipynb` and `Python/Graph Transformer Network.ipynb`.

Released dataset summary:

- number of graph samples: 6,080
- container type: Python list
- sample type: `torch_geometric.data.Data`
- key fields observed in each sample:
  - `x`
  - `edge_index`
  - `edge_attr`
  - `y`
  - `cpm_id`
  - `node_names`
  - `node_types`

Observed feature layout in the first released sample:

- `x`: node feature matrix with shape `(num_nodes, 91)`
- `edge_index`: graph connectivity with shape `(2, num_edges)`
- `edge_attr`: edge feature matrix with shape `(num_edges, 2)`
- `y`: 5-dimensional label vector

### `all_hypergraphs_with_labels-train.pkl`

This is the hypergraph training dataset used by `Python/Hypergraph Neural Network.ipynb`.

According to the notebook implementation, each hypergraph sample is expected to expose:

- `nodes`
- `edges`
- `node_features`
- `labels`

and can be converted into a sparse incidence matrix for hypergraph neural network training. Because the notebook loads this file with `torch.load(...)`, users should prepare an environment compatible with the notebook's hypergraph dependencies before inspecting it directly.
